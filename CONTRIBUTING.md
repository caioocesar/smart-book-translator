## How to Contribute

### Reporting Bugs

Include:
- OS and version (Windows 10, Ubuntu 22.04, etc.)
- Node.js version (`node -v`)
- Steps to reproduce
- Expected vs actual behavior
- Error messages or logs

### Suggesting Features

Consider:
- Is it useful for personal translation use?
- Does it maintain ethical standards?
- Is it feasible with current APIs?
- Does it improve user experience?

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test on both platforms if applicable
5. Update documentation
6. Commit with clear messages
7. Push to your fork
8. Open a Pull Request

## Code Style

### Backend (Node.js)
- Use ES6+ modules
- Async/await over callbacks
- Error handling in all async functions
- Clear variable names

### Frontend (React)
- Functional components with hooks
- Clear prop names
- Responsive design
- Accessible UI

### Documentation
- Clear explanations
- Code examples where helpful
- Update README for new features

## Testing

Before submitting:

1. **Backend**: Test all API endpoints
2. **Frontend**: Test all UI interactions
3. **Integration**: Test full translation workflow
4. **Error Cases**: Test error handling
5. **Cross-Platform**: Test on Windows and Ubuntu

## Areas for Contribution

### Nice to Have
- Dark mode

### Documentation
